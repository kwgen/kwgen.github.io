android 活动周期

发表于 2017-07-05   |   分类于 android   |  

返回栈

安卓是使用任务来管理活动的，一个任务就是一组放在栈里面的集合，启动新活动时压入栈，按下back或者finish()出栈

活动状态

  1. 运行状态：活动位于栈顶
  2. 暂停状态：活动不处于栈顶，但仍然可见
  3. 停止状态：活动不处于栈顶，又不见
  4. 销毁状态：活动从返回栈种移除

活动的生命周期

  1. onCreate()：活动的第一次创建时被调用；在这个方法种完成布局，绑定事件等操作
  2. onStart()：活动由不可见变为可见时调用
  3. onResume()：活动准备好和用户交互的时候调用，运行状态
  4. onPause()：系统准备去启动或者恢复另一个活动调用；在这个方法中将一些消耗资源释放
  5. onStop()：这个方法在活动完全不可见时调用；它和onPause()的区别，如果新活动时一个对话框式ide活动，那么onPause()会执行，onStop()不会执行
  6. OnDestroy()：在销毁之前调用，之后的活动状态为销毁状态
  7. onRestart()：活动由停止变为运行状态之前调用

生存期

  1. 完整生存期：onCreate()和onDestroy之间的经历。完成初始化和释放内存操作
  2. 可见生存期：onStart()和onStop()之间的经历。
  3. 前台生存期：onResume()和onPause()之间的经历。这时的活动是和用户交互的
